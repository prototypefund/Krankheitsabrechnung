Arbeitsumgebung

Python / PostgreSQL / SimpelMed

Benötigt werden:

  • ein Arbeitsrechner mit genügend RAM (mindestens 8 GB, besser 16 GB) und ausreichend schneller Festplatte (vorzugseise SSD),

  • eine PostgreSQL-Datenbank (»Server«), sinnvollerweise in virtueller Maschine,

  • (optional, aber sehr empfehlenswert) eine weiterere virtuelle Maschine mit der PostgreSQL-Admin-Oberfläche »pgadmin4«.
    Diese »drei« Rechner werden über ein lokales Netz verbunden und sind damit gegenseitig ansprechbar/erreichbar über TCP/IP.

Vorerst wird ausschließlich eine GNU/Linux-Umgebung verwendet, später kommt - um den Einsatz in einem realen Szenario darzustellen - ein Windows-Client hinzu.

Dieses Netzwerk kann realisiert werden über
- »Virtualbox« (GNU/Linux oder Windows) oder
- »KVM/Quemu/Virt-Manager (nur GNU/Linux).

VirtualBox oder KVM (virt-manager)

Pro Virtualbox:
Netzwerk-Bridging einfach einzustellen ohne eigene Konfiguration.

Contra:
Erheblicher Overhead mit spürbarer Herabsetzung der Ausführungsgeschwindigkeit,
(uu) langsam sowie sehr hohe Zahl an Zugriffen bei Installation des Image auf eine herkömmliche SATA-Platte bzw. bei jeder Aktualisierung
→ wenn Virtualbox, dann sollte dies zwingend auf einer SSD installiert sein!

Pro KVM:
Schneller, performanter, kein Overhead.

Contra:
Umkonfigurierung der Netzwerkeinstellungen notwendig (Bridging),
dies kan aber durch ein »einfaches (Python-)Script« durchgeführt werden.

Beide Lösungen ermöglichen auch den Datenaustausch zwischen den (virtuellen) Rechnern;
beide Lösungen simulieren den Einsatz der Software in einem Client-Server-Umfeld.

Im Folgenden werden beide Lösungen beschrieben; beide Lösungen wurden getestet.

Virtualbox

Wichtig: Beim Starten/Installieren wird als »Netzwerk« gewählt:
Adapter … Netzwerkbrücke
nicht die Versäumniseinstellung »NAT«

Als OS für die virtuellen Rechner wurden gewählt:
a) Mint (Xfce Edition https://linuxmint.com/download.php oder Debian LMDE https://linuxmint.com/download_lmde.php)
b) q4os (Aquarius Trinity https://q4os.org/downloads1.html)

Diese beiden Distributionen sind nach unserer Erfahrung schnell, genügend klein und stabil. Beide sind Debian Derivate und greifen auf die entsprechenden Repos zurück.

Anmerkung
 Grundsätzlich steht die Wahl der Linux-Distribution in Zusammenhang mit GnuMed im Ermessen des Anwenders - zu bedenken ist allerdings, dass viele Anleitungen, Manuale und auch die bisherigen Software-Pakete eine Debian-kompatible Ausstattung bevorzugen oder sogar voraussetzen.
Auch SimpelMed geht (vorerst) von Debian-Systemen (zumindest als GnuMed-Server aus), tatsächlich kann natürlich auch Windows10/11 eingesetzt werden.
